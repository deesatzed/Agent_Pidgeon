## 4. Project Specification

**Scope & goals:**
- Build a proof-of-concept for AI-to-AI (A2A) pidgin communication using Hugging Face's `hf-mount` and an MCP-style receiver
- Implement a structured pidgin message contract with handshake and resolve message types
- Create a receiver service that mounts pinned HF artifacts and resolves concept sets locally
- Support both legacy dataset-based payloads and future-facing artifact-based payloads
- Provide stdio-based sender/receiver transport via FastMCP
- Maintain backward compatibility while enabling gradual migration to the new artifact model

**Non-goals:**
- Production-grade deployment configuration
- Multi-artifact bundle support
- Complex repo-internal directory semantics
- Hard cutover from legacy `dataset_*` fields

## 7. Architecture & Environments

**High-level architecture:**

```
┌─────────────────┐
│   Sender Agent  │
│  (PidginStdio   │
│   Sender)       │
└────────┬────────┘
         │ stdio (FastMCP)
         ▼
┌─────────────────────────────────┐
│  MCP Receiver (FastMCP Server)  │
│  ┌───────────────────────────┐  │
│  │  PidginReceiverService    │  │
│  │  - handshake()            │  │
│  │  - resolve_message()      │  │
│  └───────────┬───────────────┘  │
│              │                   │
│              ▼                   │
│  ┌───────────────────────────┐  │
│  │  HfMountManager          │  │
│  │  - ensure_repo_mounted() │  │
│  └───────────┬───────────────┘  │
└──────────────┼──────────────────┘
               │ hf-mount CLI
               ▼
┌─────────────────────────────────┐
│  HF FUSE Mount                  │
│  /tmp/agent-pidgin/<repo>/      │
└─────────────────────────────────┘
```

**Module/component responsibilities and boundaries:**

- **`PidginStdioSender`** (`src/agent_pidgin/sender.py`): FastMCP client wrapper that sends handshake and resolve messages via stdio, normalizes tool call results into JSON-serializable dictionaries
- **`PidginReceiverService`** (`src/agent_pidgin/service.py`): Core business logic that handles handshake requests and resolve messages, computes effective artifact targets using precedence rules
- **`HfMountManager`** (`src/agent_pidgin/hf_mount.py`): Gateway to `hf-mount` CLI, handles mount/stop operations with idempotent behavior
- **`PidginResolver`** (`src/agent_pidgin/resolver.py`): Concept resolution logic that maps pointer names to implementation code
- **`SeedCatalog`** (`src/agent_pidgin/catalog.py`): Registry of supported concept pointers
- **`PidginConfig`** (`src/agent_pidgin/config.py`): Configuration loader that reads environment variables and provides defaults
- **`PidginHandshake` / `PidginMessage`** (`src/agent_pidgin/protocol.py`): Dataclass definitions for protocol messages with validation

**Dependency graph:**

```
PidginReceiverService
  ├─> PidginConfig
  ├─> HfMountManager
  ├─> SeedCatalog
  ├─> PidginResolver
  └─> PidginHandshake / PidginMessage

PidginStdioSender
  └─> FastMCP Client (external)

MCP App
  ├─> PidginReceiverService
  └─> FastMCP Server (external)
```
